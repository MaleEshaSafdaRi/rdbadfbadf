UI/UX Developer Interview Test

Quick Guide

  • You will receive access on a repository to be used exclusively by you.
  • You will be able to clone the repository and push new commits for 2 weeks. The exact expiration date will be confirmed when the repository URL will be provided.
  • The repository contains all the material required for the test, including an existing project with some minimal functionality.
  • You must install additional software on your local machine (see next section below).

Project Setup

  1. Download and install Git on the local machine.
  2. Clone the interview test repository locally.
    • Open a terminal window in the directory where you want to clone the repository.
    • Run command git clone `url` (the URL will be provided)
  3. Download and install NodeJS 14.17 on the local machine. If you already have another version installed, you may try to run the project without changing it to 14.17.0 but please consider that the existing project has only been tested with this version.
  4. In the cloned repository directory run the following commands in a terminal window.
    • npm install to install npm package dependencies.
    • npm run start to start the application.
  5. Navigate to http://localhost:4200 in the browser. The application should be up and running.
  6. To stop the application press Ctrl + C in the terminal window and follow confirmation instructions.
Worthy Notes:
  • The existing project was generated with Angular CLI version 13.2.3.
  • Modifying existing files would trigger a live reload of the application.
  • Adding or removing files could require a restart by stopping the application and running `npm run start` again.

Commit & Push Changes

  1. Open a terminal window in the cloned repository directory.
  2. Run git add -A
  3. Run git commit -am '[comment]'
  4. Run git push

2. Tasks

  1. Update the UI for the existing functionality on the Angular CLI project. Style, clean and refactor as needed. Allow the user to switch between two themes (light/dark). The application can be modified in any way possible if the following requirements are still fulfilled:
    • Organization
      • Must be able to select an organization.
      • Must be able to change the selected organization.
    • Departments
      • Must be able to view a list of existing departments inside the selected organization.
      • Must be able to search departments by name in the list of existing departments.
      • Must be able to view department details.
      • Must be able to edit department details.
      • Must include, as department details, a name, a manager, an annual budget and a list of employees.
    • Employees
      • Must be able to see the list of existing employees inside the selected organization.
      • Must be able to search employees by name in the list of existing employees.
      • Must be able to view employee details.
      • Must include, as employee details, the first name, the last name, the hired date, the age and the salary.
      • Could include, as employee details an image of the employee.
  2. Create story boards, mocks or other assets that help illustrate the UI/UX with the functionality to be added:
    • Must be able to add a new organization.
    • Must always have a name entered for newly added organizations.
    • Must be able to add a new department.
    • Must be able to add a new employee.
    • Must allow to the possibility to create a new organization along with its departments and their employees in one action instead of creating one level at a time. This could be associated with a wizard like control.
  3. Implement the functional UI/UX prototype for the functionality that allows the user to create a new organization with departments and employees in one action.
  4. Provide design feedback for several screenshots of complex functionality that is briefly described. Consider what is done well and what can be improved. Check next section for details.
